github pages 配置

简单介绍下所谓的部署到github上的原理：

  1. 在Github上创建的那个特别的repository（name.github.io）一个最大的特点就是其master中的html静态文件，可以通过链接http://name.github.io来直接访问
  2. hexo -g 会生成一个静态网站（第一次会生成一个public目录），这个静态文件可以直接访问
  3. 需要将hexo生成的静态网站，提交(git commit)到github上

部署的步骤如下：

  1. 建立repository
    建立和用户名对应的仓库，仓库名必须是”your_user_name.github.io” 这种固定格式

  2. hexo和仓库进行关联
    修改文件_config.yml

  3. 执行部署命令
